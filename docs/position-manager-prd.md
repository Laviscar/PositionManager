# 仓位计划小工具 PRD（更新版）

## 1. 文档目的
本 PRD 用于指导“仓位计划即时互动小软件”的设计与开发，聚焦以下目标：
- 用户仅控制**大分类仓位上限**，不固定个股目标比例。
- 用户通过输入少量关键数据，实时看到分类剩余可配仓位与计划外持仓。
- 显式展示“计划外冲动买入”资金，帮助复盘与纪律管理。

---

## 2. 产品定位与核心价值

### 2.1 定位
一个轻量、实时反馈的仓位管理工具。

### 2.2 核心价值
1. **计划前置**：先约束大类仓位比例，避免组合失衡。
2. **录入简单**：只需维护三类输入字段即可工作。
3. **偏离可视化**：自动识别计划外持仓，形成“纪律雷达”。

---

## 3. 范围定义（MVP）

### 3.1 用户手动输入项（仅三类）
1. **可用总资金**（例如：500000）
2. **大分类资金比例（%）**（例如：科技 30%、消费 20%、红利 50%）
3. **个股资金（实际持仓金额）**（例如：AAPL 30000、贵州茅台 20000）

> 说明：
> - 不提供“个股目标仓位%”配置。
> - 个股可在所属大分类内自由浮动，只受大分类上限约束。

### 3.2 系统自动项
- 每个大分类的目标金额、已用金额、剩余额度、使用率。
- 全局已配置金额、剩余可用金额、总仓位利用率。
- 自动生成并维护一个默认大分类：**exception/其他**（计划外持仓池）。

---

## 4. 关键业务规则

### 4.1 大分类规则
1. 用户手动创建的大分类，其比例总和必须等于 100%。
2. 每个大分类都有“目标金额”：
   - `分类目标金额 = 可用总资金 × 分类比例`
3. 分类已用金额为该分类下所有个股实际持仓金额之和。
4. 分类剩余额度：
   - `分类剩余额度 = 分类目标金额 - 分类已用金额`
5. 若分类剩余额度 < 0，则判定为“分类超配”，需高亮预警。

### 4.2 个股规则
1. 个股仅记录“名称/代码 + 实际持仓金额 + 所属分类”。
2. 个股**没有目标百分比**、**没有目标金额**。
3. 同一分类下允许新增、删除、编辑任意个股。

### 4.3 exception/其他（默认分类）规则
1. 系统自动创建“exception/其他”分类，用户不可删除。
2. 该分类**不需要、也不允许设置目标百分比**。
3. 用户可在此分类下手动添加个股与持仓金额。
4. 该分类用于承载计划外持仓，核心展示：
   - 计划外持仓总金额
   - 计划外持仓占可用总资金比例
5. exception/其他不参与“100%计划分类比例校验”。

### 4.4 全局一致性规则
1. 金额单位默认“元”，展示保留 2 位小数。
2. 百分比展示保留 2 位小数。
3. 当“所有分类（含 exception/其他）持仓总和 > 可用总资金”时，显示全局超配预警。

---

## 5. 信息架构与页面

## 5.1 页面 A：计划设置
### 功能
- 输入可用总资金。
- 维护大分类及其目标比例（不含 exception/其他）。
- 实时校验比例总和是否为 100%。

### 关键组件
- 总资金输入框
- 分类列表（名称、比例、自动计算目标金额）
- “新增分类”按钮
- 比例校验提示条

## 5.2 页面 B：持仓录入
### 功能
- 按分类录入个股实际持仓金额。
- 支持在每个分类下增删改个股。
- 单独显示 exception/其他分类并支持录入计划外个股。

### 关键组件
- 分类折叠面板
- 个股行（代码/名称、金额输入、删除）
- “新增个股”按钮

## 5.3 页面 C：仓位仪表盘
### 功能
- 汇总全局、分类、计划外持仓状态。

### 关键组件
1. **全局卡片**
   - 可用总资金
   - 已投入金额
   - 未投入金额
   - 总利用率
2. **分类卡片（不含 exception/其他）**
   - 目标金额、已用金额、剩余额度、使用率
   - 超配状态标识
3. **计划外卡片（exception/其他）**
   - 计划外金额
   - 计划外占比
   - 计划外个股明细入口

## 6. 计算口径（开发直接可用）

设：
- `T` = 可用总资金
- 对任一计划内分类 `c`：
  - `p_c` = 分类目标比例（0~1）
  - `target_c = T * p_c`
  - `actual_c = Σ(分类 c 下各个股实际金额)`
  - `remain_c = target_c - actual_c`
  - `usage_c = actual_c / target_c`（target_c=0 时显示 `--`）
- `actual_exception = Σ(exception/其他 下各个股实际金额)`
- `actual_total = Σ(actual_c) + actual_exception`
- `remain_total = T - actual_total`
- `usage_total = actual_total / T`（T=0 时显示 `--`）

派生指标：
- `exception_ratio = actual_exception / T`（T=0 时显示 `--`）
- 分类超配条件：`remain_c < 0`
- 全局超配条件：`remain_total < 0`

---

## 7. 交互与反馈规则

1. 任一金额或比例输入变更后，所有结果实时刷新（<200ms 感知）。
2. 输入非法值处理：
   - 非数字：阻止保存并提示
   - 负数：默认不允许；如要支持，需额外开启“融资/做空模式”（MVP 不做）
3. 颜色策略：
   - 正常：中性色
   - 接近上限（使用率 ≥ 90% 且 ≤ 100%）：黄色
   - 超配（>100%）：红色
4. exception/其他始终显示在分类列表末尾，且视觉上与计划内分类区分。

---

## 8. 数据模型（MVP）

## 8.1 实体
1. `portfolio`
   - `id`
   - `totalCapital`
   - `updatedAt`

2. `category`
   - `id`
   - `portfolioId`
   - `name`
   - `targetPercent`（exception/其他 为空）
   - `isException`（布尔）
   - `sortOrder`

3. `position`
   - `id`
   - `portfolioId`
   - `categoryId`
   - `symbol`
   - `displayName`
   - `actualAmount`
   - `updatedAt`

## 8.2 约束
- 每个 `portfolio` 必有且仅有 1 条 `isException = true` 的 `category`。
- `isException = false` 的分类其 `targetPercent` 必填。
- 计划内分类比例总和 = 100（保存时校验）。

---

## 9. 非功能要求
1. 本地数据持久化（LocalStorage/IndexedDB 任一）。
2. 首屏可用时间 < 2s（常规笔记本环境）。
3. 输入与计算需具备幂等性与可恢复性（刷新后不丢失）。

---

## 10. 验收标准（MVP）

1. 用户仅配置总资金与计划内分类比例，不配置个股目标比例，也可完成全部核心流程。
2. 系统自动存在 exception/其他分类，且不可删除、不可设置比例。
3. 用户可在 exception/其他中录入个股并看到计划外金额与占比。
4. 当某计划内分类超出目标金额时，分类明确标红预警。
5. 当总持仓超过可用总资金时，全局明确标红预警。
6. 所有金额/比例相关指标随输入实时更新。

---

## 11. 示例（基于 50 万）

- 可用总资金：500000
- 计划内分类：
  - 分类A：30%（目标150000）
  - 分类B：20%（目标100000）
  - 分类C：50%（目标250000）
- exception/其他：自动生成（无目标比例）

若当前持仓：
- 分类A下个股合计 120000 → 剩余 30000
- 分类B下个股合计 110000 → 超配 10000
- 分类C下个股合计 180000 → 剩余 70000
- exception/其他合计 20000（计划外占比 4%）

则：
- 总持仓 = 430000
- 总剩余 = 70000
- 分类B显示超配预警
- exception/其他显示“计划外 20000（4%）”
